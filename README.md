# Context Dependency in LLM-Based Requirements Clarification

This repository contains the replication package, frozen analysis dataset,
execution artifacts, annotation materials, and reproducibility notebook for the
article *A Quantitative Replication of Context Effects in LLM-Based Requirements
Clarification*.

The study investigates how different types of context affect the clarification
questions generated by a Specification-Driven Development (SDD) agent based on
Large Language Models, observed through the `/speckit.clarify` command. The
primary outcome is the number of **necessary unanswered gaps per execution**.

## Study Overview

The experiment preserves the four User Stories and six context conditions of the
reference study and adds five independent repetitions per story–condition
combination.

- **4 User Stories** (US02, US08, US19, US22).
- **6 context conditions:**
  - `C0`: User Story only;
  - `CL`: lexical context;
  - `CO`: operational context;
  - `CD`: decisional context;
  - `CS`: systemic context;
  - `CT`: all contexts combined.
- **5 independent repetitions** per User Story × condition.
- **120 valid executions** (4 × 6 × 5).
- **1,039 extracted question lines**.
- **441 internal duplicates removed**.
- **598 annotated questions**.
- **64 normalized gaps**.
- **2 independent judges** followed by adjudication.

The dependent variable, `necessary_unanswered`, counts clarification questions
that are both functionally necessary and left unanswered by the material
available to the agent.

## Frozen Analysis Dataset

The authoritative source for reproducing the statistical and descriptive results
is the frozen analysis spreadsheet:

```text
data/Matrizes de rastreabilidade - Resultado - Mestrado.xlsx
```

The reproducibility notebook recomputes every reported statistic from this
workbook. It should be treated as the authoritative analysis dataset. A copy of
the same workbook is preserved in `results/freezed-results.xlsx`, which the
notebook uses as an automatic fallback.

The workbook contains at least the following sheets:

- `user_stories`
- `materials`
- `executions`
- `raw_info`
- `judge-annotation-1`
- `judge-annotation-2`
- `Gap_normalization`
- `Adjudication`
- `Gap_convenience`

The main sheets play the following roles:

- `executions`: execution plan with run identifiers, condition, repetition,
  order, output path, and status.
- `raw_info`: extracted question lines and duplicate-control fields.
- `judge-annotation-1` and `judge-annotation-2`: the two independent annotations.
- `Gap_normalization`: codebook of normalized gaps.
- `Adjudication`: final adjudicated analysis table (the basis for all metrics).
- `Gap_convenience`: auxiliary mapping used to support gap analysis.

## Reproducing the Analysis

Install the pinned dependencies (Python 3.7.9 was used for the reported results):

```bash
pip install -r requirements.txt
```

If the project uses `uv`:

```bash
uv pip install -r requirements.txt
```

Then open and run the notebook:

```bash
jupyter notebook notebooks/analysis_reproducibility.ipynb
```

Alternatively, execute it non-interactively:

```bash
jupyter nbconvert --to notebook --execute --inplace notebooks/analysis_reproducibility.ipynb
```

The notebook:

- loads the frozen analysis spreadsheet;
- reconstructs the cleaned question set;
- recomputes judge agreement and Cohen's kappa;
- reconstructs run-level metrics;
- computes condition-level descriptive statistics;
- applies Friedman tests;
- applies paired Wilcoxon tests;
- applies Holm correction;
- computes Kendall's W and rank-biserial correlations;
- reproduces the RQ2 contrasts between CT and the isolated contexts;
- generates the category elimination matrix;
- generates the gap behavior table with eliminated, persistent, emergent,
  unchanged, and redundant gaps;
- exports CSV files to `results/`.

## Main Reproduced Results

- 120 valid executions.
- 598 annotated questions.
- 64 normalized gaps.
- Friedman test: `χ²(5) = 57.64`, `p < 0.001`.
- Kendall's W: `0.576`.
- C0 mean necessary unanswered gaps: `2.90`.
- CO mean: `0.65`.
- CT mean: `0.65`.
- In the paired RQ2 contrasts, CT was more effective than CL, CD, and CS, but did
  not differ from CO.

> The combined context condition reduced necessary unanswered gaps compared with
> several isolated contexts, but it did not produce a detectable additional gain
> over the operational context condition in this dataset.

## Reproducibility Scope

> This repository supports reproduction of the descriptive and statistical
> analyses reported in the article from the frozen analysis dataset. It also
> preserves execution artifacts and experimental materials for inspection. Full
> re-execution of the SDD agent may depend on external model availability, tool
> behavior, and execution environment.

> The frozen spreadsheet should be treated as the authoritative analysis dataset.

The notebook reproduces the analysis; it does not re-run the LLM experiment.
Re-running the agent depends on the experimental materials in `materials/` and
the preserved execution outputs in `runs/`.

## Notes on Data Interpretation

- `necessary_unanswered` is the primary dependent variable.
- A question is counted as necessary unanswered when
  `Final_Coverage == "Unanswered"` and `Final_Relevance == "Necessary"`.
- `emergent` gaps appear in a contextual condition but not in C0 for the same
  User Story.
- `eliminated` gaps appear in C0 but disappear in a contextual condition.
- `persistent` gaps continue to appear despite context.
- `redundant` gaps refer to questions whose answers were already available in the
  provided material.
